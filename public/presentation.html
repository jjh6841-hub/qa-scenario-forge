Problem

의료 EMR QA, 왜 어려운가?

⏱️

방대한 시간 소요

처방 입력·DUR 검증·청구 코드 등 수백 개 케이스를 수작업으로 작성하는 데 며칠이 걸립니다.

⚠️

전문 규정 지식 필요

심평원 DUR 지침, 의료법, 개인정보보호법 등 도메인 지식 없이는 리스크를 빠뜨리기 쉽습니다.

🔁

반복 작업 & 누락

명세서가 바뀔 때마다 시나리오를 재작성해야 하고, 경계값·예외 케이스가 누락되곤 합니다.

🤖

자동화 코드 부재

케이스를 작성해도 Playwright 코드로 옮기는 별도 작업이 또 필요합니다.

Solution

명세서 한 장 → 완성된 QA 패키지

기능 명세서를 붙여넣으면
Claude AI가 4단계 파이프라인으로
리스크 분석부터 자동화 코드까지
수 분 안에 자동 생성합니다.

각 단계는 실시간 스트리밍으로
생성 과정을 눈으로 확인할 수 있습니다.

1
리스크 분석
RiskItem[]
2
테스트 시나리오 생성
TestScenario[]
3
테스트 케이스 작성
TestCase[]
4
Playwright 코드 생성
.spec.ts

Key Features

주요 기능

🏥

의료 도메인 특화 분석

심평원·의료법·개인정보보호법 기준으로 리스크를 카테고리별로 식별하고 우선순위를 자동 산정합니다.

🗺️

리스크 히트맵 시각화

영향도 × 발생가능성 5×5 매트릭스로 리스크를 한눈에 파악할 수 있습니다.

⚡

실시간 스트리밍 UI

AI 응답이 생성되는 즉시 단계별 스켈레톤 카드가 채워지며 진행 상황을 실시간으로 보여줍니다.

🤖

Playwright 코드 자동 생성

자동화 가능한 케이스를 선별해 TypeScript 기반 E2E 테스트 코드를 즉시 생성합니다.

📦

JSON / Markdown 내보내기

생성된 결과물을 JSON 또는 Markdown 형식으로 다운로드해 팀과 공유할 수 있습니다.

🎯

3가지 프리셋 제공

처방 입력, 의사 로그인, 진료 기록 조회 등 실무 명세서 예시로 바로 체험해볼 수 있습니다.
